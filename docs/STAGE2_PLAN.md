# AutoFabric Beginner Starter — 阶段二规划说明

## 1. 阶段二目标

阶段一已经完成“本地智能工作台稳定基线”的建立。  
阶段二的目标，不再是单纯让系统跑起来，而是让这套系统开始具备：

- 更稳定的数据结构
- 更清晰的 Workspace 管理能力
- 更接近产品化的工作台体验

阶段二应聚焦于“稳定性 + 可管理性 + 产品化结构”。

---

## 2. 阶段二总体原则

阶段二推进时，遵循以下原则：

### 2.1 不再大面积边改边试
阶段一已经证明，大量零碎改动会不断制造兼容问题。  
阶段二必须按模块推进，先定边界，再实施。

### 2.2 先清理基础结构，再做交互升级
如果底层数据契约不稳定，继续堆页面会不断返工。  
因此阶段二应优先解决结构一致性问题。

### 2.3 以“Workspace”作为核心产品对象
后续所有能力，不论是后端、前端还是执行器联动，都围绕 Workspace 展开。

---

## 3. 阶段二三大方向

阶段二建议拆为三个主方向：

1. 数据契约清理
2. Workspace 管理能力
3. 产品化工作台升级

---

# 方向一：数据契约清理

## 4. 目标

彻底清理当前系统中历史遗留的混用问题，让后端返回的数据结构真正稳定。

当前阶段一虽然通过兼容逻辑让系统跑通，但仍存在以下问题：

- `next_action` 在历史数据中既可能是字符串，也可能是对象
- `current_result` 历史数据结构不完全一致
- `parser_meta` 有些记录中缺失
- 部分旧 Goal 没有 Outcome
- 前端仍需要 adapter 兼容脏数据

阶段二的目标是让这些成为“历史兼容层”，而不是长期主逻辑。

---

## 5. 方向一建议任务

### 5.1 统一 `Outcome.next_action`
目标：
- 后端内部统一为对象结构
- 数据库存储也统一为标准 JSON 结构
- 逐步减少字符串兼容逻辑

建议标准结构：

```json
{
  "summary": "string",
  "step_type": "string | null",
  "steps": [],
  "requires_human_confirmation": false
}
